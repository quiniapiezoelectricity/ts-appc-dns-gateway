# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| `main` branch | Yes |

## Reporting a Vulnerability

Do not open public GitHub issues for suspected vulnerabilities.

Use one of these channels:

1. GitHub Security Advisories: open a private report via the repository `Security` tab (`Report a vulnerability`).
2. If private advisories are unavailable, contact the maintainers directly and include `SECURITY` in the subject.

Include as much detail as possible:

- Affected component/file and deployment context
- Reproduction steps or proof-of-concept
- Expected impact (confidentiality/integrity/availability)
- Suggested mitigation (if available)

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage/update cadence: at least weekly until resolved
- Fix + disclosure timeline: coordinated with the reporter based on severity
